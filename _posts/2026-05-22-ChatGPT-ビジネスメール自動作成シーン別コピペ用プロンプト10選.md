「謝罪メール、どう書けばいいか30分悩んでる…」

そういう経験、正直けっこうある。特に感情が絡むメール——断り、催促、クレーム対応——は書き出しだけで時間が溶ける。

ChatGPTを使えば、そのストレスがほぼ消える。この記事では、**今日から使えるシーン別プロンプト10個**と、出力精度を上げる実践コツをそのまま渡す。

---

## ChatGPTでビジネスメールを作る基本の流れ

難しくない。やることは3つだけ。

**① プロンプトを入力する → ② 出力を確認・修正する → ③ 送信する**

これだけ。ただ、②を省いて③に直行する人が多い。それが一番危ない。固有名詞・金額・日付の誤りは普通に混入する。「確認してから送る」を習慣にするだけで、リスクはほぼゼロになる。

### 無料版で十分か、有料版が必要か

ぶっちゃけ、**日常的なビジネスメールなら無料版（GPT-4o mini）で十分**。

有料版（GPT-4o、月額約3,000円）が活きるのは、こういう場面だ。

- 複数の条件が絡む長文メールを一発で仕上げたい
- 微妙な温度感の調整を何度もやり直したい
- Custom Instructions（後述）をフル活用したい

普段のお礼・日程調整・社内報告なら、無料版でまったく問題ない。

### プロンプトに必ず入れる「5点セット」

これを入れるだけで、出力の質が劇的に変わる。

```
① 目的（例：商談のお礼、納期遅延の謝罪）
② 送り先（例：初対面の社外取引先、社内の上司）
③ トーン（例：丁寧・簡潔・フォーマル）
④ 含めたいポイント（箇条書きで2〜3個）
⑤ 文字数の目安（例：200字以内）
```

「なんか微妙な出力が出た」という人の9割は、この5点のどれかが抜けている。逆に全部入れると、最初の出力からほぼ使えるレベルになる。

## 【コピペOK】ビジネスシーン別プロンプト10選

以下のプロンプトは、すべて`[ ]`内を自分の情報に書き換えてそのまま使える。

---

### 1. お礼メール（商談後）

```
以下の条件でお礼メールを作成してください。
・送り先：初めてお会いした社外の方（[会社名・担当者名]）
・商談内容：[簡単な内容を1〜2行]
・トーン：丁寧かつ簡潔
・次のアクション：[例：資料送付、次回日程調整]
・文字数：200字前後
「〜させていただく」は多用しないこと。
```

**なぜ効くか：** 「〜させていただく」禁止を明示するだけで、テンプレ臭が一気に消える。

---

### 2. 依頼メール（初対面の取引先へ）

```
以下の条件で依頼メールを作成してください。
・送り先：初対面の社外取引先（[会社名・担当者名]）
・依頼内容：[具体的に]
・期日：[日付]
・トーン：丁寧・フォーマル
・文字数：250字前後
相手に負担をかけすぎない表現にすること。
```

**なぜ効くか：** 「負担をかけすぎない」の一言で、押しつけがましさが消える。

---

### 3. 断りメール（角を立てず丁重に）

```
以下の条件で断りのメールを作成してください。
・送り先：社外取引先（継続的な関係がある）
・断る内容：[例：今回の提案は見送り、イベント参加が難しい]
・理由：[簡潔に。詳細は不要でもOK]
・関係性を壊さない温度感で
・文字数：200字前後
```

**なぜ効くか：** 断りメールは「理由の書き方」が全て。詳細を書かなくていい、と指示するのがコツ。

---

### 4. 謝罪メール（納期遅延・ミス）

```
以下の条件で謝罪メールを作成してください。
・送り先：社外取引先（[会社名・担当者名]）
・謝罪の理由：[例：納品が3日遅延、資料に誤りがあった]
・再発防止策：[簡単に1〜2行]
・トーン：誠実・丁寧。過度な自己批判は避ける
・文字数：250字前後
```

**なぜ効くか：** 「過度な自己批判を避ける」を入れると、卑屈にならず誠実な文体になる。

---

### 5. 催促メール（返信・支払いの確認）

```
以下の条件で催促メールを作成してください。
・送り先：[社外取引先 or 社内担当者]
・催促の内容：[例：先週送った見積もりへの返信、請求書の支払い確認]
・前回の連絡からの経過：[例：1週間]
・トーン：丁寧だが、はっきりと用件を伝える
・文字数：150字前後
```

**なぜ効くか：** 催促は「はっきり伝える」と明示しないと、ぼんやりした文になりがち。

---

### 6. 日程調整メール（複数候補を提示）

```
以下の条件で日程調整メールを作成してください。
・送り先：社外取引先（[会社名・担当者名]）
・目的：[例：オンライン打ち合わせ、訪問商談]
・候補日時：[例：〇月〇日14時、〇月〇日午前中]
・所要時間：[例：1時間程度]
・文字数：200字前後
```

**なぜ効くか：** 候補日時を箇条書きで渡すと、見やすい形に整えてくれる。

---

### 7. 社内への報告・共有メール

```
以下の条件で社内向け報告メールを作成してください。
・送り先：[上司 or チーム全体]
・報告内容：[例：今週の進捗、トラブルの発生と対応状況]
・トーン：簡潔・ファクトベース
・余分な前置きは不要
・文字数：150〜200字
```

**なぜ効くか：** 「余分な前置き不要」で、社内メールらしいテンポが出る。

---

### 8. 自己紹介メール（異動・着任挨拶）

```
以下の条件で着任挨拶メールを作成してください。
・送り先：[社外の既存取引先 or 社内関係部署]
・自分の情報：[前任の部署・新しい役職・担当業務]
・引き継ぎ担当者：[名前]
・トーン：丁寧・親しみやすい
・文字数：200字前後
```

**なぜ効くか：** 引き継ぎ先の名前を入れると、具体性が上がって実用的な文になる。

---

### 9. 提案・売り込みメール（新規営業）

```
以下の条件で新規営業メールを作成してください。
・送り先：初対面の企業担当者
・自社・自分の情報：[会社名・担当者名・提供サービス]
・相手へのベネフィット：[例：コスト削減、業務効率化]
・アクション：[例：15分のオンライン面談を依頼]
・トーン：押しつけがましくなく、簡潔に
・文字数：200字前後
```

**なぜ効くか：** 「押しつけがましくなく」を入れると、読んでもらえる温度感になる。

---

### 10. クレーム対応メール（顧客への返信）

```
以下の条件でクレーム対応メールを作成してください。
・送り先：不満を持つ顧客
・クレームの内容：[例：商品の不具合、サービス対応への不満]
・対応方針：[例：返金対応、交換対応、再発防止策の説明]
・トーン：誠実・落ち着いた口調。感情的にならない
・文字数：300字前後
```

**なぜ効くか：** 対応方針を先に決めてから入力するのがポイント。ChatGPTに方針まで考えさせると、事実と違う内容が混入するリスクがある。

## 出力精度を上げる3つのコツと、やってはいけないこと

### コツ① Custom Instructionsに自分の情報を登録する

ChatGPTの「カスタム指示」機能に、以下を登録しておくと毎回の入力が楽になる。

```
・役職：[例：営業部・係長]
・業種：[例：IT系SaaS企業]
・よく使う敬称・NGワード：[例：「〜させていただく」多用禁止]
・文体の方向性：[例：簡潔・ファクトベース]
```

設定後は「私は〇〇部の〇〇です」を毎回書かなくて済む。文体の一貫性も保たれる。

### コツ② ゼロから生成より「下書きのリライト」が精度高い

意外と知られていないが、これが最も実用的な使い方だ。

```
以下のメールをビジネス向けに丁寧に整えてください。
元の意図・事実関係は変えないこと。
[自分が書いた下書きをここに貼る]
```

ゼロから作ると「言いたいことと微妙にズレた文」が出やすい。下書きを渡す方が、意図のズレが圧倒的に少ない。

### コツ③ 3パターン出力で「選ぶだけ」にする

```
上記のメールを3パターン（丁寧・簡潔・フレンドリー）で出力してください。
```

この一行を追加するだけで、状況に応じて選べる選択肢が手に入る。思考コストがゼロになる。

---

### やってはいけない3つのこと

**機密情報・個人情報をそのまま入力しない。**
顧客名・金額・社外秘の情報は仮名・ダミーに置き換えて使う。企業によっては社内規定で禁止されているケースもある。

**ファクトチェックを省かない。**
日付・金額・担当者名の誤りは普通に起きる。送信前に必ず確認する。

**プロンプトを長くしすぎない。**
「長い方が精度が上がる」は誤解。構造化された短い指示の方が、出力がブレない。
